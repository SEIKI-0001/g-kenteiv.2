# G検定 合格ナビ

G検定受験者が毎日使う学習サポート画面のモックです。今日やること、復習すること、弱点、今週の学習計画を確認できます。

## 今回のコンテンツ拡充

- G検定の試験範囲をもとに、弱点チェックの単元を8領域へ拡張
- 今日の学習を12本へ拡張
- 復習カードを9本へ拡張
- すぐ確認できる選択問題を26問へ拡張
- 弱点カードに確認トピックを追加
- 学習タスク、復習カード、クイック診断から単元別の確認問題を出せるように変更
- 復習カードの「覚えていた」「曖昧」「忘れた」で次回表示日の見え方が変わるように変更
- 画面ロジックと学習コンテンツを分離し、`data/content.js` に集約
- 本番移行用のデータ構造メモを `docs/content-model.md` に追加
- 学習進捗を `localStorage` に保存し、リロード後も続きから見られるように変更
- 状態管理の本番移行メモを `docs/state-model.md` に追加
- Supabase Auth/RLS と連携し、ログイン中の進捗をユーザ単位で同期

## 画面の考え方

- 今日やる量を少なく見せる
- 何を押せば学習が始まるかを分かりやすくする
- 間違えた問題を次の復習につなげる
- バッジは「できるようになったこと」の記録として見せる
- 休んだ後に戻ってこられる導線を残す

## 画面

- ホーム: 今日のおすすめ学習、次にやること、最近のつまずき
- 今日の学習: 思い出し練習、演習、説明、復習
- 復習: 自信度を選びながらカードを戻す
- 弱点: 単元ごとの弱点理由と次の行動
- バッジ: できるようになったことの記録
- 学習計画: 今週優先する単元と学習リズム

## 次の段階

- 第5段階: 問題ごとの回答履歴、正答率、弱点推移を保存する
- 第6段階: ローカルモックからNext.js/DB付きの本番構成へ移行する

## ファイル構成

- `index.html`: 画面の土台
- `app.js`: 画面描画、タブ切替、復習結果、確認問題の操作
- `data/content.js`: 学習タスク、単元、復習カード、バッジ、確認問題
- `styles.css`: UIスタイル
- `scripts/build-env.js`: Vercel環境変数からブラウザ用設定を生成
- `supabase/schema.sql`: Supabase SQL Editorに登録するDBスキーマ
- `docs/content-model.md`: 本番実装に向けたコンテンツ構造メモ
- `docs/state-model.md`: 学習状態の保存構造メモ

## Supabase連携

Vercelの環境変数に以下を設定します。

```txt
SUPABASE_URL=
SUPABASE_ANON_KEY=
```

`SUPABASE_SERVICE_ROLE_KEY` はブラウザに出さないため、このアプリでは設定しません。
Vercelのビルド時に `npm run build` が `env.js` を生成し、画面側からSupabaseへ接続します。

## ローカル確認

```bash
npm run build
python3 -m http.server 4174
```

ブラウザで `http://127.0.0.1:4174/` を開いて確認できます。
